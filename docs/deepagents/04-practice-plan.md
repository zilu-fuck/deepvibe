# 04. 实践与练习建议

## 学习目标

如果你把 `deepagents` 当成学习项目，建议把目标定成下面四个层次：

1. 会跑
2. 会读
3. 会改
4. 会自己做一个简化版

## 第一阶段：会跑

建议动作：

1. 安装依赖
2. 跑一个最小示例
3. 确认模型配置和基本调用链跑通

完成标准：

- 能成功调用 `create_deep_agent`
- 能让 Agent 处理一个简单任务

## 第二阶段：会读

建议动作：

1. 画出模块关系图
2. 记录 `graph -> middleware -> backend` 主链
3. 分清 SDK、CLI、evals、examples 的边界

完成标准：

- 能用自己的话解释这个项目结构
- 能指出核心入口文件和关键目录

## 第三阶段：会改

建议练习：

1. 修改 system prompt 或 profile
2. 增加一个简单自定义工具
3. 调整一个 middleware 的行为
4. 给某个 example 改一个小能力

完成标准：

- 改动后能跑通
- 知道自己改动影响的是哪一层

## 第四阶段：会自己做一个简化版

练习题建议：

1. 用 LangGraph 手写一个最小 Agent
2. 只保留规划、文件读取、命令执行三个能力
3. 不做 CLI，先做 Python 调用版
4. 最后再尝试加一个子代理能力

完成标准：

- 你能复刻它的核心思想，而不是记住它的文件名

## 推荐实践题

### 练习 1：读主线

任务：

- 读 `create_deep_agent`
- 写一页自己的结构总结

目标：

- 建立主骨架

### 练习 2：改文件系统能力

任务：

- 找出文件系统 middleware 的工具注入方式
- 理解 `read_file`、`write_file`、`edit_file`、`execute` 是怎么注册进去的

目标：

- 理解工具系统和 backend 关系

### 练习 3：看一个完整示例

任务：

- 选 `deep_research` 或 `text-to-sql-agent`
- 写出它依赖了哪些能力模块

目标：

- 建立“抽象到应用”的联系

### 练习 4：理解 CLI 产品层

任务：

- 看 `libs\cli\deepagents_cli\app.py`
- 只关注会话、消息、审批、模型切换这四类逻辑

目标：

- 理解 SDK 如何被包装成产品

## 最后建议

学习这个项目时，最好不要一上来就追求“全部细节都懂”。

更有效的顺序是：

1. 先抓主架构
2. 再抓关键模块
3. 然后做一次小修改
4. 最后再回头读复杂细节

这样会比一开始就陷进 `app.py` 或所有 examples 更轻松。
